# A股量化交易系统多端界面开发规范（Codex执行版）

你现在要开发的是一个 **A股量化交易系统的多端 Web 界面**，同时支持：

- PC Web 端
- 平板端
- 移动端

这不是官网，不是通用 SaaS 后台，不是原生桌面客户端，也不是单纯把 PC 页面缩小到手机里。

你的任务是设计并实现一个 **现代化、专业化、深浅主题、适配多设备的量化交易 Web 应用**。

---

## 1. 产品定位

系统面向以下用户：

- 量化研究员
- 交易员
- 风控人员
- 运维监控人员

核心使用目标：

- 查看策略状态
- 分析收益与回撤
- 查看持仓、委托、成交
- 监控风险与告警
- 快速执行关键操作
- 在不同设备上都能清楚获取核心信息

---

## 2. 多端设计总原则

必须严格遵守以下原则：

### 原则 1：同一产品，多端分层
- PC、平板、移动端属于同一产品
- 视觉风格、主题 token、组件语言必须统一
- 但信息密度、布局方式、交互方式必须按设备调整
- 绝对禁止把 PC 页面简单缩放到移动端

### 原则 2：PC 完整，移动端聚焦
- **PC Web 端承担完整业务功能**
- **平板端承担浏览、监控、轻操作**
- **移动端承担查看、告警、快速处置，不承担重度复杂操作**

### 原则 3：交易场景优先
界面必须符合量化交易系统习惯，而不是普通互联网后台习惯：

- 专业
- 清晰
- 稳定
- 高效
- 可扫读
- 风险感知强

### 原则 4：现代 Web 应用风格
- 是现代 Web 应用，不是老式券商软件
- 也是浏览器产品，不要做成假装原生客户端
- 保留专业感，但不能笨重、过时、压抑

---

## 3. 设备策略

### PC Web 端
适用场景：

- 长时间使用
- 高频操作
- 深度分析
- 回测分析
- 多模块联动
- 表格和图表并重

PC 端必须具备完整功能：

- Dashboard 总览
- Strategy Detail 策略详情
- Backtest 回测分析
- Live Monitor 实盘监控
- 委托与成交
- 持仓与账户
- 风控中心
- 日志中心
- 系统设置

### 平板端
适用场景：

- 巡检
- 浏览
- 轻量分析
- 快速确认状态
- 轻度管理操作

平板端要求：

- 保留核心页面
- 适当减少并列区域
- 图表和表格可以纵向堆叠
- 支持常用筛选和切换
- 支持关键操作，但不承载特别复杂的批量管理

### 移动端
适用场景：

- 随时查看
- 接收告警
- 看收益和状态
- 看持仓、订单、风险
- 快速启停或紧急处理

移动端要求：

- 只保留高频核心信息
- 不做复杂多表联动
- 不做高复杂度回测参数编辑
- 不做超重型日志分析
- 重点支持：
  - 总览
  - 告警
  - 策略状态
  - 收益
  - 持仓
  - 委托/成交摘要
  - 快速控制

---

## 4. 响应式断点要求

建议至少支持以下断点：

- Mobile: `< 768px`
- Tablet: `768px ~ 1279px`
- Desktop: `>= 1280px`

要求：

- Desktop：完整多栏布局
- Tablet：减少横向并列，优先双栏或上下布局
- Mobile：单栏纵向布局，底部导航或顶部切换

不得出现：

- 文字过小
- 表格完全不可读
- 图表被压扁
- 操作按钮挤在一起
- 多层滚动区域混乱

---

## 5. 总体视觉要求

### 风格关键词
- 专业
- 现代
- 克制
- 高级但不炫技
- 数据产品感
- 长时间使用舒适

### 禁止项
- 大面积渐变
- 重阴影
- 夸张圆角
- 花哨动画
- 营销官网风
- 过度留白
- 大量装饰性图标
- 为了“现代感”牺牲信息密度
- 为了“适配移动端”把专业度做没了

---

## 6. 主题要求

必须支持：

- 深色主题
- 浅色主题

其中：

- 默认深色主题
- 浅色主题完整适配
- 所有端统一 token 体系
- 所有组件必须同时支持深浅主题

### 深色主题要求
- 不是纯黑
- 使用深灰、深蓝灰、石墨色体系
- 长时间观看不刺眼
- 盈亏与告警色清楚但不荧光

### 浅色主题要求
- 使用冷白、浅灰、近白色体系
- 不要发黄
- 靠边框和层级，而不是重阴影

---

## 7. 信息架构原则

### PC 端信息架构
采用：

- Topbar
- Sidebar
- Content
- 可选右侧 Drawer / Detail Panel

### 平板端信息架构
采用：

- 顶部导航或压缩侧边导航
- 内容区纵向堆叠
- 主要信息块按优先级排列

### 移动端信息架构
采用：

- 顶部标题栏
- 底部导航或顶部 Tab
- 单栏主内容
- 卡片 + 列表 + 小型图表 + 简化表格

移动端不要照搬 PC 左侧边栏。

## 11. 表格与列表策略

### Desktop
优先使用 DataTable，支持：
- 固定表头
- 排序
- 筛选
- 列显隐
- 横向滚动
- 行 hover
- 行选中
- 数值右对齐

### Tablet
可以保留简化表格，但应减少列数，保留关键字段。

### Mobile
不要强行使用完整宽表。
移动端优先改造成：
- 卡片列表
- key-value 列表
- 可展开详情
- 紧凑列表

### 数值规则
- 盈利：success
- 亏损：danger
- 中性：默认文本
- 不能只靠颜色表达，必须保留正负号和标签说明

---

## 12. 图表策略

### Desktop
支持完整图表交互：
- tooltip
- 时间切换
- 筛选
- 联动

### Tablet
保留核心图表，减少并列数量。

### Mobile
只保留最关键趋势图：
- 收益趋势
- 风险趋势
- 简化分布图

移动端图表必须：
- 清晰
- 不拥挤
- 不塞太多维度

---

## 13. 交互规则

### Desktop
- 高频操作 1~2 次点击完成
- 抽屉、Tab、行内操作优先
- 避免全屏弹窗

### Tablet
- 控制层级
- 保持足够点击热区
- 复杂操作需分步

### Mobile
- 单手可操作优先
- 关键按钮明显
- 避免密集小按钮
- 关键控制要有确认
- 复杂编辑拆成单独页面或步骤流

所有端都必须：
- 操作有明确反馈
- 危险操作二次确认
- 错误提示明确
- 空状态友好
