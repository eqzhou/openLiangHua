# OpenLiangHua 用户使用手册

更新时间：`2026-04-29`

## 1. 适用范围

本手册只说明当前仓库里已经实现、并且能在代码与现有产物中互相印证的能力。

当前正式对外应以 `React + FastAPI` 版本为主：

- React 工作台：`http://127.0.0.1:5174`
- FastAPI 接口：`http://127.0.0.1:8989`
- Streamlit：`http://127.0.0.1:8501`

其中 `Streamlit` 仍然保留，但按照当前架构决议，它是回退面，不应再作为主操作界面继续扩展。

## 2. 系统现在能做什么

当前版本已经实现的核心能力如下：

- 查看研究总览、因子截面、模型回测、候选股、持仓、AI 分析、数据状态、页面服务状态。
- 保存研究参数，包括训练区间、验证区间、测试区间、标签周期、候选股数量。
- 手工触发研究动作：
  - 刷新部分面板
  - 重建特征与标签
  - 运行岭回归基线
  - 运行梯度提升树基线
  - 运行自适应融合策略
  - 生成 AI 研判摘要
  - 生成最新未标注截面推理
- 在 Web 界面中查看和维护持仓池/观察池，查看实时快照、历史验证排名、最新推理排名、分批观察计划、历史评分曲线。
- 在持仓页中生成最新盯盘清单和最新操作备忘，并回写到页面。
- 查看 AI 双池对照：
  - 历史验证池
  - 最新推理池
- 在数据管理页查看本地数据状态，触发 `Tushare` 增量刷新或全流程刷新，并查看/手动触发可选 `MyQuant` 数据任务。
- 在服务页手工刷新实时行情快照，并查看 `FastAPI + React/Vite + PM2 + Streamlit fallback` 的只读健康状态与日志。
- 本地单管理员可在登录后修改自己的密码。

## 3. 当前版本不应对外承诺的能力

以下能力在当前仓库中没有形成可发布的 Web 功能，不应写进正式对外说明：

- 不支持券商接口下单。
- 不支持自动交易或自动执行买卖。
- 不支持完整的多用户管理后台、用户 CRUD 或 RBAC；当前是本地单管理员登录、会话校验、退出登录和改密码。
- 不支持在 Web 界面里配置完整 `MyQuant` 参数矩阵；当前只提供状态检测和基于现有配置的手动任务入口。
- 服务页不提供 Web 启停/重启服务按钮；它是只读健康大盘，避免扩大命令执行面。

## 4. 启动与访问

### 4.1 推荐启动方式

在仓库根目录执行：

```bash
pm2 start ecosystem.config.cjs
```

默认会启动：

- API：`8989`
- React：`5174`

常用命令：

```bash
pm2 status
pm2 logs openlianghua-api
pm2 logs openlianghua-web
pm2 restart openlianghua-api
pm2 restart openlianghua-web
pm2 stop ecosystem.config.cjs
pm2 delete ecosystem.config.cjs
```

### 4.2 分开启动

启动 API：

```bash
bash scripts/start_research_api.sh
```

启动 React：

```bash
bash scripts/start_react_web.sh
```

### 4.3 登录说明

- 页面右上角提供登录入口。
- 未登录时系统处于只读模式。
- 登录后才能执行写操作，例如保存研究参数、触发研究动作、刷新实时快照、生成盯盘清单、生成操作备忘。
- 登录账户来自服务端认证配置，不应把默认开发口令当成正式发布口令继续使用。
- 登录后可在右上角修改当前管理员密码；修改需要输入旧密码，新密码至少 8 位并同时包含字母和数字。

## 5. 推荐日常使用流程

建议按照下面顺序使用系统：

1. 打开首页，先看服务状态、行情快照覆盖率、风险提醒、重点持仓与候选。
2. 如需执行动作，先登录。
3. 进入“数据管理”，确认日线、特征、标签的最新日期是否正常；如需补数，再做 `Tushare` 增量或全流程刷新，或按需执行可选 `MyQuant` 手动任务。
4. 进入“工作台”，确认研究参数是否正确；如需调整训练区间、标签周期或候选股数量，在这里保存。
5. 在“工作台”执行研究动作，通常顺序为：
   - 刷新部分面板
   - 重建特征与标签
   - 岭回归
   - 梯度提升树
   - 自适应融合
   - AI 研判摘要
   - 最新未标注截面推理
6. 进入“研究概览”“模型回测”“候选股”“AI 分析”检查结果。
7. 进入“持仓”页面查看实时价格、推理排名、分析状态，并按需生成盯盘清单与操作备忘。

## 6. 页面说明

### 6.1 首页 `/`

首页是当前主操作入口，主要用于快速看值班信息：

- 服务状态与行情快照状态
- 风险提醒
- 重点持仓
- 重点候选
- 持仓表、候选表、AI 推理表
- 快捷动作

首页当前快捷动作以“最新未标注截面推理”和“生成 AI 研判摘要”为主。

### 6.2 工作台 `/workspace`

工作台是主要执行面，包含三类能力：

- 研究参数保存
- 研究动作执行
- 页面缓存清理、分享链接复制、界面偏好重置

工作台里的研究动作已经接到后端模块，不是静态按钮。

### 6.3 研究概览 `/overview`

研究概览用于看整体状态，当前已实现内容包括：

- 模型横向比较
- 净值曲线
- 数据快照
- 最佳模型摘要

适合先判断当前哪套模型表现最好，再决定是否深入回测页。

### 6.4 因子探索 `/factors`

当前可用于：

- 查看最新因子截面排名
- 查看缺失率
- 查看单只股票的因子快照
- 查看单因子历史序列

页面支持表格预设、列管理、排序、列拖拽和宽度调整。

### 6.5 模型回测 `/backtests`

当前可用于：

- 查看验证集与测试集表现
- 查看净值曲线、区间收益、风控前后对比
- 查看月度、年度、状态分段诊断
- 查看特征重要度
- 查看稳定性摘要

### 6.6 候选股 `/candidates`

当前可用于：

- 查看某模型、某分段下的候选股总表
- 查看单只股票的评分历史
- 从候选股页面跳转到 AI 分析或持仓页面
- 复制股票代码
- 复制当前视图链接

### 6.7 持仓 `/watchlist`

这是当前最接近日常交易辅助的页面，但仍然是“辅助决策页”，不是下单页。

当前已实现：

- 按范围筛选观察池
- 添加、编辑、删除持仓池/观察池条目
- 按推理排名、历史排名、浮亏比例、参考市值排序
- 拉取并显示实时快照
- 查看参考价、最新价、浮盈亏、历史验证排名、最新推理排名
- 查看多轮 AI 研讨状态与结论
- 查看分批观察/减仓计划
- 查看历史评分曲线
- 查看最新 AI Shortlist
- 生成最新盯盘清单
- 生成最新操作备忘

生成后的盯盘清单与操作备忘会写入 `reports/weekly/`，并在页面中直接展示。

观察池维护规则：

- `ts_code + type` 是条目的身份键，创建后不通过编辑表单修改；如需换代码或类型，应删除后重新添加。
- 编辑入口用于更新名称、成本、股数、备注等人工维护字段。
- Web 维护走 `/api/watchlist-config/items`，后端按当前登录用户隔离数据。

### 6.8 AI 分析 `/ai-review`

当前页面是“双池对照”结构：

- 最新推理池
- 历史验证池

每一池当前都可以查看：

- 候选总表
- 当前选中股票的核心论点
- 公告/新闻/研报摘要
- 分析概览
- 外部模型摘要（若已生成）
- 完整字段表

这个页面适合判断“历史验证观点”和“最新推理观点”是否一致。

### 6.9 数据管理 `/data`

当前页面可用于：

- 查看日线、特征、标签面板的存在性、行数、股票数、最新日期
- 查看 `.env` 是否存在
- 查看 `TUSHARE_TOKEN` 是否已配置
- 查看当前实际落地数据源与配置数据源是否一致
- 触发 `Tushare` 增量刷新日线
- 触发 `Tushare` 全流程刷新
- 查看 `MYQUANT_TOKEN`、`gm` SDK、最近 `MyQuant` 日线数据状态
- 手动触发 `MyQuant` 下载、清洗增强、研究面板重建

注意：

- 当前 `config/universe.yaml` 的正式主数据源是 `tushare`。
- `MyQuant` 是可选数据源入口，只使用现有配置和可选截止日期，不在 Web 中暴露完整下载参数矩阵。
- 如果系统当前主要使用的是别的数据产物，页面会提示“配置源”和“实际落地源”不一致。
- 当登录用户观察池为空时，市场日线主表刷新会返回业务提示，要求先添加观察标的。

### 6.10 页面服务 `/service`

当前页面可用于：

- 查看最近实时快照状态
- 手工刷新实时快照
- 查看 `FastAPI`、`React/Vite`、`PM2`、`Streamlit fallback` 状态
- 查看端口、进程、最近状态字段和日志

注意：

- 该页是只读健康大盘，不提供启动、停止或重启按钮。
- 它适合做“React/API 是否在线”“PM2 是否可见”“行情快照是否可用”“回退服务是否还活着”的检查。

## 7. 结果产物在哪里看

当前系统的主要结果产物落在以下位置：

- `reports/weekly/`
  - 回测指标
  - 候选结果
  - AI 摘要
  - 最新推理结果
  - LLM 请求与响应摘要
  - 盯盘清单
  - 操作备忘
- `data/staging/`
  - 日线与事件缓存
- `data/features/`
  - 特征面板
- `data/labels/`
  - 标签面板

当前仓库中已经可以看到以下真实产物类型：

- `tushare_overlay_latest_brief.md`
- `tushare_overlay_inference_brief.md`
- `000078_watch_plan_2026-04-17.md`
- `000078_action_memo_2026-04-17.md`

## 8. 使用边界与发布口径建议

正式发布时，建议统一使用下面这套口径：

- 这是一个 A 股研究与持仓辅助系统。
- 它支持研究数据检查、模型训练与回测查看、候选股筛选、持仓观察、AI 双池分析、盘前清单和操作备忘生成。
- 它当前不直接连接券商，不自动交易，不替代人工下单。
- 它支持 Web 持仓池/观察池维护，但 `ts_code + type` 作为身份键不可编辑。
- 数据刷新页以 `Tushare` 为当前主配置，同时提供可选 `MyQuant` 状态检测和手动任务入口。
- 服务页是只读健康大盘，不是远程运维控制台。

如果对外说明超过以上边界，就容易写到仓库里还没有真正交付的能力。
